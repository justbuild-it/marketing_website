# Iriscale Application Assessment

This directory contains the assessment framework and documentation for the Iriscale application, supporting the creation of training materials, videos, marketing content, and promotional assets.

## Directory Structure

```
iriscale_app_assessment/
├── README.md                 # This file
├── assessment_plan.md        # Master plan and methodology
├── feature_inventory.md      # Checklist tracking all 24 modules
├── templates/
│   └── feature_template.md   # Reusable template for documenting modules
├── modules/                  # Individual module documentation
│   └── (one file per module)
├── screenshots/              # UI captures organized by module
│   └── (organized by suite/module)
└── deliverables/             # Final outputs
    ├── training/
    ├── videos/
    ├── marketing/
    └── promo/
```

## Quick Start

### 1. Review the Plan
Start with `assessment_plan.md` to understand the methodology and scope.

### 2. Check Progress
Use `feature_inventory.md` to see which modules have been documented.

### 3. Document a Module
1. Copy `templates/feature_template.md` to `modules/[module_name].md`
2. Explore the module in the live application
3. Take screenshots and save to `screenshots/[suite]/[module]/`
4. Fill in all sections of the template
5. Update `feature_inventory.md` to mark progress

### 4. Create Deliverables
Once modules are documented, create final assets in `deliverables/`.

## Content Tags

When documenting, tag content for its intended use:

| Tag | Purpose | Output |
|-----|---------|--------|
| `TRAINING` | Step-by-step guidance | User guides, onboarding docs |
| `VIDEO` | Visual demonstration | Tutorial scripts, demo flows |
| `MARKETING` | Value communication | Website copy, feature pages |
| `PROMO` | Attention-grabbing highlights | Social posts, one-pagers |

## File Naming Conventions

- **Modules:** `[suite]_[module_name].md` (e.g., `vision_telescale.md`)
- **Screenshots:** `[module]_[description].png` (e.g., `telescale_main_dashboard.png`)
- **Deliverables:** `[type]_[topic].[ext]` (e.g., `training_getting_started.md`)
