# Iriscale Application Assessment Plan

## Purpose
Systematically document the Iriscale application to support:
- **Training Documentation** - User guides, onboarding materials, how-to articles
- **Training Videos** - Screencast scripts, demo sequences, tutorial outlines
- **Marketing Website** - Feature descriptions, value propositions, use cases
- **Promotional Material** - One-pagers, social content, sales decks

---

## Assessment Methodology

### Phase 1: Application Discovery
1. **Access & Navigation Audit**
   - Document login/authentication flow
   - Map main navigation structure
   - Identify all accessible modules/screens
   - Screenshot key interfaces

2. **Architecture Mapping**
   - Identify the 5 main suites and their relationships
   - Document module hierarchy within each suite
   - Map data flow between components

### Phase 2: Feature Documentation
For each module, capture:
- **What it does** (functionality)
- **How to use it** (workflow/steps)
- **Why it matters** (value proposition)
- **Who benefits** (persona/use case)

### Phase 3: Content Tagging
Tag each documented feature for intended use:
- `[TRAINING]` - Suitable for how-to guides
- `[VIDEO]` - Good visual demonstration potential
- `[MARKETING]` - Strong value prop for website
- `[PROMO]` - Headline-worthy differentiator
